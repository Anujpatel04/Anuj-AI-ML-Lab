# Contributing

## Quick Start

1. Fork the repository
2. Create a branch: `git checkout -b feature/your-feature-name`
3. Make your changes
4. Submit a pull request

## Guidelines

- **Code**: Follow PEP 8, include docstrings, update `requirements.txt`
- **Structure**: Each project needs `README.md` and `requirements.txt`
- **Environment**: Use root `.env` for API keys, don't commit `venv/` or `myenv/`
- **PRs**: Update docs, test changes, use clear commit messages

## What We Welcome

- New AI agents, bug fixes, documentation improvements, code refactoring, feature enhancements

## What Not to Include

- API keys, virtual environments, large binaries, generated files (`__pycache__`, `.pyc`)
